We use a native input[type="file"] component to upload a file, and it can also take a picture without requesting your camera permission on mobile.
So the behavior of the input[type="file"] is quite different between mobile and desktop. Try opening this page on both mobile and desktop to see the difference.



You'll find:
  • It can only upload a file on desktop
  • It can upload a file on mobile and also take a photo, whether it's the front or rear camera
  • Interactive logic is different, UI is different

So if a selfie(both face detection or face liveness) is not required on the mobile, the native input component is enough.
